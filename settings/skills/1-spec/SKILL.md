# 1 Spec

Invoke `spec-driven-development`.

Use this as the first explicit workflow step for new projects, features, significant changes, or unclear requirements.

Follow `spec-driven-development` for clarification, section order, and style.
Output to `SPEC.md` or the existing project spec location.

Stop after the spec for non-trivial work unless the user explicitly asks to continue.
