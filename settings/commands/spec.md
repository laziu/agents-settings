description: Start spec-driven development before code.
---

Invoke `spec-driven-development`.

Clarify:
- Objective and target users
- Goals, non-goals, and success metrics
- Core features and acceptance criteria
- Functional and non-functional constraints
- Stack preferences and constraints
- Boundaries: always / ask first / never

Produce `SPEC.md` with a compact PRD-shaped spec: objective, users/use cases, requirements, stack, commands, structure, style, testing, boundaries, success criteria, and open questions.

Confirm with the user before implementation.
